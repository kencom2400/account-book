# Epic Issue List for Account Book Application

このドキュメントは、資産管理アプリケーションの開発において作成するEpic（大項目）の一覧です。

## Epicとは

Epicは、複数の関連するIssue（Story/Task）をまとめた大きな機能単位です。
本プロジェクトでは、機能要件（FR）や非機能要件（NFR）、開発タスクをEpicとして管理します。

## Epic管理のルール

### Epic Issue の作成

1. **タイトル形式**: `[EPIC] <Epic名> - <簡潔な説明>`
2. **ラベル**: `epic`, 対応する機能ラベル（例: `integration`, `classification`）、優先度ラベル
3. **マイルストーン**: 対応するPhaseのマイルストーンを設定
4. **説明**:
   - Epicの目的と概要
   - 関連するIssueのリスト（チェックボックス形式）
   - 受入基準
   - 完了条件

### サブタスクの紐付け

- 各IssueのDescriptionに「**Epic**: #<Epic番号>」を追記
- Epicの説明文に関連Issueをチェックボックスで列挙

### Epic の進捗管理

- Epic配下の全Issueが完了したらEpicをClose
- 進捗率は手動で更新（例: 3/10 Issues completed）
