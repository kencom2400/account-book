# Issue作業完了報告テンプレート

このテンプレートは、Issue作業完了時にGitHub Issueへコメントを投稿する際に使用します。

---

## 📝 報告フォーマット

```markdown
## 🎉 作業完了報告

Issue #XXX「[Issue タイトル]」の作業が完了しました。

---

## 📊 実施した作業

### [作業項目1]

**内容**:

- [詳細内容]

**成果物**:

- [ファイル1]
- [ファイル2]

**コミット**: `[コミットメッセージ]`

---

### [作業項目2]

（同様に記載）

---

## 📂 成果物

### 新規作成ファイル（合計Xファイル）

1. **[カテゴリ1]** (Xファイル)
   - `path/to/file1`
   - `path/to/file2`

2. **[カテゴリ2]** (Xファイル)
   - `path/to/file3`

### 更新ファイル（合計Xファイル）

1. **[ファイル名]**
   - `path/to/file`
   - [変更内容の概要]

---

## ✅ 達成した目標

### Issueの受入基準

- [x] [受入基準1]
- [x] [受入基準2]
- [x] [受入基準3]

### 期待される効果

✅ **[効果1]**

- [詳細]

✅ **[効果2]**

- [詳細]

---

## 🚀 次のステップ

### 1. レビュー依頼

- [ ] [レビュー項目1]
- [ ] [レビュー項目2]

### 2. マージ後

- [ ] [実施事項1]
- [ ] [実施事項2]

---

## 📊 作業時間

- [作業項目1]: 約X時間
- [作業項目2]: 約X時間
- **合計**: 約X時間

---

## 🔗 関連リンク

- **PR**: [PR URL または「作成予定」]
- **ブランチ**: `feature/issue-XXX-description`
- **コミット数**: X

---

## 📝 備考

### [追加情報のタイトル]

[追加情報の内容]

---

以上で、Issue #XXXの作業が完了しました。レビューをお願いします。
```

---

## 🔧 実装方法

### GitHub CLIを使用した自動投稿

```bash
# Issue番号を変数に設定
ISSUE_NUMBER=XXX

# コメント本文をヒアドキュメントで変数に格納
BODY=$(cat <<'EOF'
## 🎉 作業完了報告
...（上記テンプレートの内容）
EOF
)

# GitHub CLIでコメント投稿
gh issue comment $ISSUE_NUMBER --body "$BODY"
```

---

## 📐 必須項目

### 1. タイトル

明確な見出し（「作業完了報告」等）

### 2. 実施した作業

具体的な作業内容（Phase分けされている場合は各Phase）

### 3. 成果物

作成・更新したファイル一覧（カテゴリ別に整理）

### 4. 達成した目標

受入基準の達成状況と期待される効果

### 5. 次のステップ

レビュー依頼や今後の作業

### 6. メタ情報

作業時間と関連リンク（PR、ブランチ、コミット）

---

## 💡 ベストプラクティス

### 1. 明確で簡潔に

- 要点を絞って記載
- 箇条書きを活用
- 専門用語は必要最小限

### 2. 視覚的に分かりやすく

- 見出しを適切に使用（##、###）
- 絵文字を活用（📊、✅、🚀など）
- チェックボックスで進捗を明示

### 3. 追跡可能に

- コミットハッシュやメッセージを記載
- PRへのリンクを含める
- 関連Issueへのリンク

### 4. 次のアクションを明確に

- レビュー依頼を明示
- マージ後のタスクをリストアップ
- 必要なフォローアップを記載

---

## 🚫 避けるべきこと

### 1. 情報不足

❌ 「作業完了しました」だけ
✅ 具体的な作業内容と成果物を記載

### 2. 過度に詳細

❌ すべてのコード変更を列挙
✅ 主要な変更点に絞る

### 3. 曖昧な表現

❌ 「いくつかのファイルを更新」
✅ 「3つのファイルを更新：file1.ts, file2.ts, file3.ts」

### 4. 次のステップなし

❌ 報告だけで終わる
✅ レビュー依頼や今後の作業を明記
