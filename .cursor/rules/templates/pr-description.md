# Pull Request説明テンプレート

このテンプレートは、Pull Request作成時の説明文に使用します。

---

## 📝 PR説明フォーマット

````markdown
## 📋 概要

[WIP/完成] [機能名/修正内容]の実装

## 🎯 関連Issue

Closes #XXX
Related to #YYY

## 📊 変更内容

### [カテゴリ1]

- [変更内容1]
- [変更内容2]

### [カテゴリ2]

- [変更内容3]

## ✅ 実装状況

- [x] 基本実装完了
- [x] ユニットテスト追加
- [x] E2Eテスト追加
- [x] ドキュメント更新
- [ ] 統合テスト作成中

## 🧪 テスト結果

### ローカル実行

- ✅ **Build**: 成功
- ✅ **Lint**: エラーなし
- ✅ **Unit Test**: 全テストパス (XX tests)
- ✅ **E2E Test**: 全テストパス (XX tests)

### 実行コマンド

```bash
./scripts/test/lint.sh
./scripts/test/test.sh all
./scripts/test/test-e2e.sh frontend
```
````

---

## 🔧 実装方法

### GitHub CLIを使用したPR作成

```bash
# 通常PRを作成（--draftフラグは使用しない）
gh pr create \
  --title "<type>(<scope>): <subject>" \
  --body "$(cat .github/PULL_REQUEST_TEMPLATE.md)"
```

**重要**: PRは常に通常PRとして作成します。ドラフトPR（`--draft`フラグ）は使用しません。

---

## 📐 必須項目

### 1. 概要

何を実装/修正したか簡潔に説明

### 2. 関連Issue

`Closes #XXX`, `Related to #YYY` で関連Issueを明記

### 3. 変更内容

カテゴリ別に変更内容を列挙

### 4. 実装状況

チェックリストで進捗を明示

### 5. テスト結果

ローカルでの実行結果を記載

### 6. 変更ファイル

新規作成と更新ファイルを分けて記載

### 7. レビュー観点

重点確認項目と注意点を明記

---

## 💡 ベストプラクティス

### 1. WIP/完成を明示

タイトルに[WIP]または[完成]を付ける

### 2. 具体的な変更内容

「いくつかの修正」ではなく、具体的な変更を記載

### 3. テスト結果は必須

ローカルでの実行結果を必ず記載

### 4. レビュアーを明示

適切なレビュアーを@メンションで指定

---

## 🚫 避けるべきこと

### 1. 情報不足

❌ 「実装しました」だけ
✅ 何を、なぜ、どのように実装したか記載

### 2. テスト結果の省略

❌ 「テストは通りました」だけ
✅ 具体的なテスト結果を記載

### 3. 関連Issueの記載漏れ

❌ Issue番号の記載なし
✅ Closes #XXX で明記
