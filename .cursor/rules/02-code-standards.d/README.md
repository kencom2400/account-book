## 使用方法

このディレクトリのファイルは、`@inc-all-rules`で自動的に読み込まれます。

**読み込み順序（重要）**:

- 優先度の高いルール（`01-XX`）が後から読み込まれます
- 読み込み順序: `04-XX`（メモ）→ `03-XX`（一般）→ `02-XX`（優先）→ `01-XX`（必須）
- **理由**: 後から読み込まれたルールが優先されるため、矛盾する内容がある場合に優先度の高いルールが優先される

個別に参照する場合は、以下のように指定してください：

```markdown
参照: `.cursor/rules/02-code-standards.d/01-01-type-safety.md`
```

## 新しいルールファイルの追加

新しいルールファイルを追加する場合は、**`00-ADDING-NEW-RULES.md`**を参照してください。

**基本的な方針**:

- **優先度に応じた適切な番号を付ける**（最重要）
- 新しいルールの優先度を正しく評価し、適切な位置に挿入
- 既存ファイルの番号を変更するのは、優先度を正しく反映するための当然の作業
- `README.md`の「ファイル一覧と優先順位」セクションを必ず更新

**避けるべきこと**:

- 優先度が高いルールを末尾に追加すること（優先度の意味を無視すること）
